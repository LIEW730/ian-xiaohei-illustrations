# 卡皮巴拉学编程 - Codex 完整 Prompt

## 直接使用的 Codex Prompt

复制下面的内容到 Codex，执行生成：

```
Use $ian-xiaohei-illustrations 为下面这个故事生成 4 张配图。

故事主题：有一只卡皮巴拉正在学习编程。它不断地努力，试错，最后成功成为了伟大的程序员。

核心要求：
- 16:9 横版、纯白背景、黑色手绘线稿、细线、轻微抖动
- 大量留白，主体只占画面 40%-60%
- 少量红色、橙色、蓝色中文手写批注（不超过 3 个字）
- 用卡皮巴拉（温和、圆润、疗愈感的大型啮齿动物）替换小黑角色
- 每张图只表达一个核心动作、状态或情感转变
- 卡皮巴拉必须参与核心动作，不能只是装饰
- 怪诞、有创意、清爽，但不幼稚、不卖萌

## Shot List（4 张图的详细规划）

### 图 1：热情开始
**放在故事的位置**：故事开头

**主题**：卡皮巴拉充满热情和好奇心地开始学习编程

**核心意思**：一切的开始都来自于热情和渴望

**结构类型**：状态/情感转变

**卡皮巴拉的动作**：
- 身体向前倾，眼睛明亮、充满期待
- 用前爪轻轻按在电脑键盘上
- 表情应该是认真但轻松的

**视觉元素**：
- 电脑屏幕上有闪闪发光的代码光芒
- 卡皮巴拉周围有温暖的气氛
- 可以加入简单的光线暗示"开始"

**建议中文标注**：
- 蓝色标注："开始"或"热情"
- 或红色标注："Come on"

---

### 图 2：困惑试错
**放在故事的位置**：中间段落 - 开始遇到难题

**主题**：卡皮巴拉在代码面前感到困惑和挫折

**核心意思**：学习的过程中必然会遇到障碍和困难

**结构类型**：状态/困境

**卡皮巴拉的动作**：
- 头歪向一侧，露出困惑的表情
- 用前爪挠头或挠脸，做出思考状
- 身体略微后退，表现出退缩感
- 眼睛可以画成大问号的形状

**视觉元素**：
- 屏幕上显示红色的错误代码或报错信息（ERROR、BUG 等）
- 代码框上飘出红色的 × 符号或问号
- 卡皮巴拉周围可以有乱七八糟的代码碎片

**建议中文标注**：
- 红色标注："卡住了" 或 "Bug"
- 蓝色标注："为什么？"

### 图 3：不断尝试
**放在故事的位置**：中间-后期段落 - 多次试错和坚持

**主题**：卡皮巴拉从失败中站起来，一次次重新尝试

**核心意思**：坚持和重复是学习的本质，每一次失败都是前进

**结构类型**：流程/循环/隐喻

**卡皮巴拉的动作**：
- 卡皮巴拉在画面中出现多个"幻影"状态：一个摔倒、一个站起、一个再次开始

- 前爪可以做出"重新开始"的手势

**视觉元素**：
- 多层代码框，逐渐从错误变成正确
- 可以用楼梯或上升的曲线表现进度
- 红色的失败印记逐渐转变成蓝色的成功迹象
- 卡皮巴拉周围应该有"努力"和"坚持"的视觉暗示

**建议中文标注**：
- 橙色标注："又来一次" 或 "重试"
- 蓝色标注："更近了"
- 绿色标注（如果可以）："进步"

**核心意思**：所有的努力和坚持都会有回报，成功是水到渠成的结果

- 表情应该是平静而满足的，甚至可以有微妙的骄傲感
- 前爪可以做出"完成"或"胜利"的手势

**视觉元素**：
- 背景应该有成功的视觉表现：金光、光芒、完整的代码框

## 生成提示

1. 每张图应该是独立的、可单独理解的
2. 4 张图连起来应该能讲述一个完整的故事弧线
3. 卡皮巴拉在每张图中的情感应该逐步变化：热情 → 困惑 → 坚定 → 满足
4. 中文标注词应该简洁有力，3 个字以内最佳
5. 留白很重要，不要把画面填满 - 让观看者的眼睛有休息的地方

## 质量检查清单

生成后请检查：
- ✅ 背景是否纯白（不要米色、纸纹、阴影、渐变）
- ✅ 线稿是否细黑、有轻微抖动感
- ✅ 主体是否只占画面 40%-60%（大量留白）
- ✅ 中文标注是否清晰、不超过 3 个字
- ✅ 卡皮巴拉是否真正参与了核心动作（不是只站在旁边）
- ✅ 是否避免了 PPT 感和过度装饰
- ✅ 4 张图是否讲述了完整的学习→成功故事
```

## 使用方法

1. **复制上面 Codex Prompt 代码块中的所有内容**
2. **打开你的 Codex 环境**
3. **粘贴 Prompt**
4. **执行生成**
5. **等待 4 张图完成**

Codex 会自动：
- 理解故事的 4 个阶段
- 为每个阶段生成对应的配图
- 确保卡皮巴拉的表情和动作逐步推进
- 生成纯白背景、黑色手绘线稿、红橙蓝批注的最终图片

**预计生成时间**：3-5 分钟（取决于你的 Codex 配置）
